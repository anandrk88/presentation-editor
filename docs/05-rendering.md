## The geometry evaluator (the engine room)

PowerPoint's 187 preset shapes are defined in the spec as tables of **adjust
values** (avLst), **guide formulas** (gdLst), and **path commands** whose
arguments reference guides. `presetGeom.ts` evaluates those tables at the shape's
current size:

```

  Ōö£ŌöĆ look up the preset definition (vendored from the spec ŌåÆ src/render/presetDefs.json)
  Ōö£ŌöĆ evaluate guide formulas (a tiny interpreter: */ +- ?: abs at2 cos sin sqrt min max ŌĆ” )
  ŌööŌöĆ emit EvaluatedPath[] { d, noFill, noStroke } honoring multi-path, even-odd holes,
     arcŌåÆellipse conversion, and lighten/darken sub-fills
```

Because it evaluates rather than hard-codes paths, every shape renders correctly
at **any aspect ratio**, and **adjust values from imported files round-trip**
(corner radius, arrowhead depth, callout pointer, ŌĆ”).

The 158 KB `presetDefs.json` is generated by `scripts/gen-presets.mjs` from a
vendored copy of the spec's `presetShapeDefinitions.xml`; re-run it to refresh.

## Charts

`ChartView` renders column, bar, line, area, scatter, radar, pie, and doughnut ŌĆö
each with grouping variants (clustered/stacked/100%), markers, smoothing, data
labels, error bars, legend positions, axis titles, chart/plot area fill+border,
gridline color/visibility, and per-element text styling (`partStyles`).

`chartLayout(shape)` computes the plot rectangle and margins; `chartPartRegions(shape)`
returns clickable boxes for the title/axes/legend/labels so the canvas can offer
[per-element selection](08-features.md#charts).

## One renderer, three consumers

- **Canvas** (`EditorCanvas`) renders at full size with a hit/selection overlay.
- **Thumbnails** (`SlidePanel`) render small and are **memoized** per slide so a
  commit only re-renders the slide that changed (big-deck performance).
- **Present mode** renders full-screen.

Because they share `SlideView`, they never drift.
